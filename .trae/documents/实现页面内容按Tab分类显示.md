# 实现Tab内容懒加载组件化

## 需求分析
用户希望保持单页面应用结构，通过底部导航栏的tab切换来显示不同内容，但要求通过懒加载组件的形式实现，以避免代码集中度过高和一次性加载所有内容的问题。

## 实现计划

### 1. 创建组件文件
在components目录下创建以下组件：
- `FundTab.tsx`：基金相关内容（市场概览、基金持仓、基金估值）
- `MetalTab.tsx`：贵金属相关内容
- `MarketTab.tsx`：市场相关内容（宏观经济）

### 2. 实现懒加载
在主页面中使用React的`lazy`和`Suspense`实现组件按需加载：
```tsx
const FundTab = React.lazy(() => import('./components/FundTab'));
const MetalTab = React.lazy(() => import('./components/MetalTab'));
const MarketTab = React.lazy(() => import('./components/MarketTab'));
```

### 3. 修改主页面渲染逻辑
- 保留现有的`activeTab`状态管理
- 使用条件渲染根据`activeTab`的值显示对应的懒加载组件
- 添加`Suspense`组件处理懒加载时的加载状态

### 4. 拆分内容
将原首页的内容拆分到对应组件：

#### FundTab组件
- 包含"市场概览"部分
- 包含"基金持仓"部分
- 包含"基金估值"部分

#### MetalTab组件
- 包含"贵金属"部分

#### MarketTab组件
- 包含"宏观经济"部分

### 5. 数据管理
- 保持现有的数据获取逻辑在主页面
- 通过props将数据传递给各个子组件
- 确保数据更新时子组件能够及时响应

### 6. 样式和布局
- 保持现有的样式和布局不变
- 确保在不同屏幕尺寸下的响应式布局正常工作
- 确保组件切换时的过渡流畅

### 7. 优化用户体验
- 添加适当的加载状态提示
- 确保切换tab时数据立即可用
- 保持各个板块的样式一致性

## 技术实现
- 使用React.lazy和Suspense实现组件懒加载
- 使用React的状态管理管理当前激活的tab
- 通过props传递数据和回调函数
- 保持现有的样式和布局结构

通过这个实现，用户可以通过底部导航栏的tab切换来查看不同类别的内容，同时通过懒加载组件的方式减少初始加载时间和代码复杂度，使页面结构更加清晰和易于维护。
