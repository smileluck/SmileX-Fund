# 移除基金份额和成本价相关逻辑，统一采用持仓金额和盈亏计算

## 问题分析
当前代码中存在基金份额(shares)和成本价(costPrice)的相关逻辑，需要移除这些逻辑，统一采用持仓金额和持仓盈亏/盈亏计算的方式。

## 修改计划

### 1. 修改 `app/page.tsx` 文件
- **修改 `calculateHoldingData` 函数**：
  - 移除 `shares` 和 `costPrice` 变量的计算
  - 简化函数逻辑，只使用持仓金额和盈亏进行计算
  - 更新函数返回的 UserHolding 对象结构

- **修改 `handleAddHolding` 函数**：
  - 移除 `shares` 和 `costPrice` 参数的处理
  - 只使用 `holdingAmount` 进行持仓金额的计算

- **修改 `handleBatchAddHolding` 函数**：
  - 移除 `shares` 和 `costPrice` 参数的处理
  - 只使用 `holdingAmount` 进行持仓金额的计算

### 2. 检查 `app/components/AddHoldingModal.tsx` 文件
- 确认文件中已正确使用持仓金额和盈亏，无需修改

### 3. 检查 `app/components/FundTab.tsx` 文件
- 确认接口定义中没有 `shares` 和 `costPrice` 相关参数
- 确认组件使用的是正确的持仓金额和盈亏计算方式

### 4. 检查 `lib/mockData.ts` 文件
- 确认 `UserHolding` 接口中没有 `shares` 和 `costPrice` 字段
- 确认其他相关接口定义正确

## 修改要点
1. 确保所有函数和接口定义中移除 `shares` 和 `costPrice` 相关逻辑
2. 确保只使用持仓金额和盈亏进行计算
3. 确保修改后的代码逻辑正确，不影响现有功能

## 预期结果
修改完成后，系统将不再使用基金份额和成本价的概念，而是统一采用持仓金额和持仓盈亏/盈亏计算的方式，使代码更加简洁和一致。
